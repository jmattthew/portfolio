Styling of fields

Float Labels

The "float labels" style has been adopted in Google's Material Design standard because it more clearly separates fields, uses space more efficiently, and visually groups together each field's information: label, user input, placeholder message, and error message.

Automatically Formatting

Automatically formatting input as users type helps ensure valid input and is easier for users. For example, the "phone" field here immediately erases letters & special characters on keyup, leaving only valid numerals. Since it automatically adds the "( ) -" characters, users don't need to learn any required formatting. Users also know immediately that they won't get an error message from leaving out hidden requirements.

Optional fields

Labeling fields as "optional" rather than as "required" creates less visual clutter because almost all fields are required. This labeling also emphasizes the benefits to the user rather than the hurdles.

Large format

Large elements, especially radio & check-box buttons are both easier to see and easier to tap.

Feedback for users

Instant positivity

Positive feedback is welcome and not distracting. It tells users that they can continue but don't need to.

Delayed negativity

Negative feedback is stressful and distracting. Negative feedback on keyup may be inappropriate because we don't know if the user has finished editing the field. Similarly, negative feedback on blur needlessly distracts users from filling out the next field and feels like micromanagement. Users sometimes skip or leave fields incomplete for valid reasons, intending to return before submitting. By tapping to submit, the user signals to us that they're ready for their input to be evaluated.

Co-located

As Google's Material Design standard observes, feedback is easier to find when it originates from where the user is currently looking: the element they're editing or tapping when feedback arrives. Since negative feedback arrive on the tap of submit, the error message begins with the submit buttons and cascades upwards, leading the user's eye to the top of the form and the first field-specific error.

Soft

Adding very brief transitions between every state gives a feeling of gentleness and elegance. Even the hover states transition over 100 milliseconds.
