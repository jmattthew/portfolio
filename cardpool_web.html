Background

After launching Cardpool's mobile app, my next project was to revamp its website. The existing website had not been designed for mobile devices, had an out-dated appearance, and a checkout flow that was not optimized. We knew that although many customers accessed the website from tablet and mobile devices, those devices had poor conversion to sales. Therefore, it was critical for the new UX to provide an easier-to-use shopping cart and checkout flow for all device sizes. My designs met all of these goals, and you can view the final product at Cardpool.com.

Process

While still finishing the native mobile app project, I began working with the product manager on our strategy for the website project. I helped the PM define the time-line for launching high-level features while I also began UX research. The marketing team at Cardpool's parent company had recently completed new branding guidelines and a new color palette, which I then incorporated into the style guide documentation. After my sitemap was approved, I began creating wirefames & prototypes to use for conducting live usability tests. With the prototypes validated through testing, I proceeded to create the final pixel-perfect mockups and UX documentation for the engineering team. This included micro-interactions, transitions, & animations, error states, and website micro-copy.

Research

View PDF

My research included analysis of direct competitors in gift card exchange (e.g. Raise.com) as well as best practices from the eCommerce industry (e.g. Amazon). Additionally, I had a chance to iterate on the mobile app by incorporated the qualitative feedback & user metrics that we had received since the launch of the mobile app. Based on this research, the PM and I determined and scoped several new page-level features to incorporate.

Style Guide

The style guide I created goes beyond typography, color, and layout. It also defines every state of interactivity of each screen element. Here you can see an excerpt of the form element documentation.

Due to the emphasis on improving the checkout flow, I wanted form elements to be best in class. Using the new "float label" standard employed by Material Design, form elements are color branded, and feature unique color states and animated transitions for hover, focus, valid, invalid, and other states.

Usability testing

Tap to open prototype

I created the prototype of sample form fields to justify the approach to stakeholders and provide exact transition and timing effect details to engineering. The handling of form validation and error states that is detailed in the prototype served to increase conversion during checkout, which is inherently prone to user error.

Next, it was critical to perfect the page that allowed users to select a set of gift cards to add to their carts. While my innovation of using the slider feature was successful for mobile devices, user feedback and competitive analysis indicated that a traditional product list would be more successful for the website.

I created several working prototypes that contained real product data, and developed a set of tasks for usability testing subjects. The prototypes included a version of the page with a slider, like the mobile app, and a Kayak-like version in which the user could reduce a long list of cards by using several filters. Testing revealed that, unlike mobile users, desktop users preferred to see a long list of options. Additionally, they found Kayak-like filters to be overwhelming, and preferred to manually sift through their options.

Transitions & Animations

Tap to open live website

Where budget allowed, I argued for the live site to incorporate transitions & animations to add to user delight. Sometimes transitions should be as subtle as 50 millisecond hover effects. Sometimes they should be dramatic, such as with the presentation of the card list shown here. The cards slide from off-screen to their resting position using a staggered animation effect that I designed.

Usability testing of this effect validated this approach and its added expense. Users reported excitement and increased attention.
